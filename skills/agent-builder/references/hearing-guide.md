# ヒアリングガイド

一度に2〜3問ずつ、会話として聞く。回答から次の質問を選ぶ。**STEP 1〜4はすべて聞き切ること**（会話の中ですでに答えが出ている項目だけは聞き直さなくてよい）。ヒアリングの深さがそのまま完成品の質になる — ここで手を抜くと、あとで何度も作り直すことになる。

## STEP 1: その人自身を知る（役割・知識の材料）

- お仕事は何をされていますか？（業種・立場・一人か組織か）
- 普段の1日はどんな流れですか？
- 仕事でよく作るもの・よく書くものは何ですか？（見積書、SNS投稿、報告書、コード…）

初心者向けの聞き方: 「新しく秘書さんを雇うとしたら、まず何を教えますか？その内容をそのままエージェントに教えます」

## STEP 2: 任せたいことを知る（役割・自動化の材料）

- 毎日・毎週、繰り返しやっている作業は何ですか？
- 「これ誰かやってくれたらなあ」と思う作業は？
- 苦手なこと・時間を食っていることは？
- 決まった時間にやることはありますか？（朝のメールチェック、月末の請求書…）

聞き分けのコツ: **繰り返し＋手順が決まっている**→スキル化・自動化候補。**毎回判断が要る**→エージェント本体の役割に。

## STEP 3: 情報と道具を知る（道具・安全の材料）

- 仕事の情報はどこにありますか？（PC内フォルダ、Google Drive、メール、会計ソフト…）
- 普段使っているツール・サービスは？
- 扱う情報に機密なもの（顧客情報・財務・健康情報）はありますか？ → あれば安全設計を厚くする。医療・金融・法律なら規制要件も確認

## STEP 4: 好みを知る（役割の仕上げ・記憶の材料）

- 報告は結論だけ派？経緯も知りたい派？
- 口調の好みは？（丁寧・フランク・方言など、実際の会話サンプルがあると最良）
- どこまで勝手に進めてよくて、どこから確認してほしいですか？
- エージェントに名前をつけますか？（愛着は継続利用の最大の推進力。軽く提案してよい）

## ヒアリング完了の確認（構築前に必ず）

STEP 1〜4を聞き終えたら、次のサマリー表を見せて認識合わせをする。ユーザーが「合ってる」と言うまで設計図フェーズに進まない：

| 聞いたこと | ユーザーの答え（要約） |
|---|---|
| 仕事・立場 | |
| 任せたいことトップ3 | |
| 繰り返し・定時の作業 | |
| 情報のありか・使うツール | |
| 機密データの有無 | |
| 口調・報告スタイルの好み | |
| 勝手に進めてよい範囲／確認が要る範囲 | |

空欄がある行は、埋まるまで追加で聞く。

## ヒアリングの禁じ手

- 専門用語で聞かない（「MCPは何を繋ぎますか」ではなく「メールやカレンダーもAIに見えるようにしたいですか」）
- 一度に5問以上並べない
- 相手の答えを設計に翻訳して復唱する（「つまり毎朝の売上確認を自動化したい、ということですね」）— 認識ズレをここで潰すと後工程が全部楽になる
