# 6要素の実装ガイド

エージェントの実体は「設定ファイルの組み合わせ」。ここでは各要素をClaude（Claude Code / デスクトップアプリ / Cowork）でどう実装するかをまとめる。環境により細部が変わるので、機能の有無に迷ったら公式docs（https://code.claude.com/docs/）をWebFetchで確認する。

## 1. 役割 — CLAUDE.md（中核・必ず作る）

置き場所: プロジェクト直下の `CLAUDE.md`（その作業場所専用）または `~/.claude/CLAUDE.md`（全体共通）。毎セッション自動で読み込まれる＝エージェントの人格と前提知識になる。

### CLAUDE.mdテンプレート

```markdown
# （エージェント名）の心得

## あなたの役割
（例: 私は◯◯業を営む△△の秘書として、日々の事務・調査・文書作成を補佐する）

## ユーザーについて
- 仕事: （業種・立場・扱う商品やサービス）
- よく使う言葉: （業界用語・社内用語の対訳）
- 大事にしていること: （品質観・顧客観）

## 仕事のルール
- （例: 金額・日付は必ず元データを確認してから書く）
- （例: 顧客名の入った文書を作ったら送信前に必ず見せる）
- （例: 迷ったら選択肢を並べず、推奨案を1つ出して理由を添える）

## 口調・報告スタイル
- （例: 結論から。詳細は聞かれたら。敬語は不要）

## やってはいけないこと
- （例: ◯◯フォルダのファイルは読むだけ。変更しない)
- シークレット（パスワード・APIキー）をファイルに直書きしない
```

書き方の勘所: **ルールは「なぜ」が想像できる粒度で少なく**。20行の生きたルールは200行の死んだルールに勝る。運用しながら足し、月1で減らす。

## 2. 知識 — CLAUDE.md＋参照ファイル

少量（〜50行）ならCLAUDE.mdに直接。多い場合（価格表・業務マニュアル・過去事例）は別ファイルにして、CLAUDE.mdから「◯◯のときは `docs/price-list.md` を読む」と参照させる。全部を毎回読み込ませない（コストと精度の両方に悪い）。

手順が決まった繰り返し作業は**スキル**（`~/.claude/skills/<名前>/SKILL.md`）に切り出す。スキル化の基準: 「月3回以上やる」×「手順を口で説明できる」。スキル作成はskill-builderスキルに委ねてよい。

## 3. 道具 — MCP接続とツール権限

- ユーザーが使う外部サービス（Gmail・カレンダー・Drive・Slack・会計ソフト等）はMCPコネクタで接続できる場合がある。**公式・信頼できる提供元のみ**。接続時は「AIに何が見えるようになるか」を必ず説明する
- ローカルのフォルダ構成が仕事の中心なら、CLAUDE.mdに「どこに何があるか」の地図を書くだけでも道具になる

## 4. 記憶 — メモリ運用

環境に自動メモリ機能があればそれを使う。なければファイルで自作できる:

- `memory/MEMORY.md` に「1行1事実」で追記（好み・決定事項・進行中案件）
- CLAUDE.mdに「セッション開始時にmemory/MEMORY.mdを読む」「新しい決定・好みを聞いたら追記する」と書く

記憶に入れるのは**次回も使う事実だけ**（今回の作業ログは入れない）。肥大化したら月1で整理する。

## 5. 自動化 — 定期実行とフック

- 「毎朝◯◯をまとめる」「毎週金曜にレポート」→ 環境の定期実行機能（スケジュールタスク／cron）で設定
- 「◯◯したら必ず△△する」という機械的な決まり事 → フック（hooks）設定が向く。ただしシェルコマンドを実行する場合はインジェクションリスクを確認する
- 自動化は**手動で3回うまくいった作業だけ**を対象にする。いきなり自動化しない
- 発展形として、定期実行の先に「ループ設計」（目標だけ渡し、エージェントが実行→検証→修正を達成まで自分で回り続ける方式。2026年に体系化された手法）があるが、動き続ける＝コストも動き続けるので、上級者が・上限（最大ターン数・時間・予算）を決めた上でだけ検討する。初心者にはまず定期実行で十分

## 6. 安全 — 権限設計

- 読み取り専用でいい範囲、書き換えていい範囲、触ってはいけない範囲を分けてCLAUDE.mdに明記する
- 破壊的操作（削除・上書き・外部送信・支払い）は「必ず確認」をルール化
- 確認スキップ系設定（`--dangerously-skip-permissions`等）は使わない。楽になるのは事故の直前まで
- 機密データ（医療・金融・法律）を扱う場合は、データの保存場所・アクセス範囲を最初に設計し、規制要件（GDPR/HIPAA等）に触れる

## 発展: メインセッション用エージェント定義（Claude Code上級者向け）

Claude Codeでは、エージェント定義ファイル（`~/.claude/agents/<名前>.md`）を `claude --agent <名前>` でメインセッションとして起動できる。`model`・`tools`・`initialPrompt`（起動時に自動送信される最初の指示）・`memory` などをフロントマターで固定でき、「起動したら即、昨日の続きから始まる相棒」が作れる。フィールド仕様はsubagent-builderスキル（入っていれば）の `references/frontmatter-spec.md` と共通。無い環境では公式docs（https://code.claude.com/docs/en/sub-agents）をWebFetchで参照する。
