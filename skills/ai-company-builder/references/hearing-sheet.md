# ヒアリングシート（Phase 1用）

一度に2〜3問ずつ、会話として聞く。答えから次の質問を選ぶ。**A〜Eの全セクションを聞き切ること**（会話の中ですでに答えが出ている項目だけは聞き直さなくてよい）。ここは設計全体の土台 — 棚卸しが浅いと、活用マップも組織図も砂上の楼閣になる。

## 0. ゴールの言語化（最初に必ず・ここを飛ばさない）

AI化は手段であって目的ではない。最初に「何のためか」を言葉にしてもらう。ここが曖昧なままだと、活用マップもロードマップも「便利だけど何のためだったか分からない物」になる。

- 1年後、何がどうなっていたら「AIを入れてよかった」と言えますか？（数字で言えるとなおよい：週◯時間浮く、売上◯％、外注費◯円削減…）
- ※時間軸は本人に合わせる。3年後・5年後の構想を持っている人はその長期ゴールも聞き、「長期ゴール→1年後の中間ゴール→最初の90日」と段階を分けて物差しにする
- 一番変えたいのはどれですか？ **時間**（自分の時間を取り戻す）／**お金**（利益・コスト）／**気持ち**（嫌な作業から解放される・不安を減らす)
- 理想の1日を教えてください。AIが完璧に働いてくれてる日、あなたは何をして過ごしていますか？

**ゴールが出てこない人への逆質問**（「わからない」と言われたらこちらから）：
- 「今の仕事で、一番嫌いな時間はいつですか？ **それが消えた状態**を最初のゴールにしましょう」
- 「もし週に丸1日自由な時間が増えたら、何に使いますか？ それがゴールの正体です」

決まったゴールは以降の全工程の物差しにする（Phase 2のマップの「効果」はこのゴールへの貢献度で測り、Phase 4のロードマップはこのゴールからの逆算で組む）。

## A. 事業の姿

- どんな事業をされていますか？（業種・商品/サービス・お客さんは誰か）
- 規模は？（一人 / 家族経営 / 従業員◯人）
- 売上はどこから生まれていますか？（主力の商流を言葉で）

## B. 時間の使い方（業務の棚卸し）

- 1週間の時間の使い方をざっくり教えてください（接客◯割、事務◯割、制作◯割…）
- その中で「自分がやらなくてもいい気がする仕事」は？
- 繰り返しの多い仕事トップ3は？

棚卸しが苦手な相手には、この表を渡して埋めてもらう：

| 業務 | 週あたり時間 | 定型度（毎回同じ手順？） | 自分がやる必要ある？ |
|---|---|---|---|

## C. 痛み

- 今いちばんのボトルネックは何ですか？（時間・人手・品質・集客…）
- 外注していること・外注したいことは？（外注できる仕事はAI化の有力候補）
- 過去にITツール導入で挫折した経験は？あれば原因は？（同じ轍を踏まない設計にする）

## D. データと道具

- 仕事の情報はどこにありますか？（紙 / Excel / 会計ソフト / メール / 頭の中）
- 「頭の中」の割合が高い場合：AI化の前に言語化が必要。その工程もロードマップに入れる
- 顧客情報・財務・健康情報など機密データはありますか？ → あれば守り（Phase 5）を厚くする

## E. 予算とリスク許容度

- AIに月いくらまでかけられますか？（ツール課金の現実的な上限）
- 失敗が許されない業務はどれですか？（→そこは人間の確認を必ず挟む設計に）
- AIの利用経験は？（ChatGPT触ったことある程度 / 日常的に使う / ほぼ未経験）→ ロードマップの傾斜を決める

## ヒアリング完了の確認（Phase 2に進む前に必ず）

A〜Eを聞き終えたら、次のサマリー表で認識合わせをする。ユーザーが「合ってる」と言うまでPhase 2に進まない：

| 聞いたこと | ユーザーの答え（要約） |
|---|---|
| 1年後のゴール（時間/お金/気持ち） | |
| 事業・規模・お客さん | |
| 時間の使い方（棚卸し結果） | |
| いちばんの痛み・外注したいこと | |
| データのありか・使っている道具 | |
| 機密データの有無 | |
| 予算の上限・失敗が許されない業務 | |
| AI利用の経験値 | |

空欄がある行は、埋まるまで追加で聞く。

## 聞き方の注意

- 「AIで何がしたいですか」とは聞かない（相手はAIで何ができるか知らないから答えられない）。**仕事の話だけを聞き、AIへの翻訳はこちらの仕事**
- 数字（時間・件数・金額）はできるだけ具体的に取る。Phase 2のマップとPhase 4の効果測定の精度が全部ここで決まる
